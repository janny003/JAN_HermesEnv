# LAND8116: 인코딩 일괄 변환 회귀와 복구 (2026-05)

상황 요약:
- 타이머 다이얼로그 미표시 이슈 수정 중, 일부 파일이 UTF-8로 보이는 상태에서 인코딩 정규화 시도가 섞이며
  `C2001 (상수에 줄 바꿈 문자)`, `C1057 (매크로 확장에서 예기치 않은 파일 끝)`, user-defined literal 관련 구문 오류가 연쇄 발생.
- 특히 CP949 기반 레거시 파일에 대해 "전체 트리 일괄 변환"은 고위험.

재현 신호:
- 문자열 리터럴이 실제 개행으로 깨짐:
  - `TextOut(_T("Manual TIMER closed`
  - 다음 줄에 `"), ... )`
- 동일 시점에 한글 주석/리터럴 모지바케 동반.

안전 복구 절차:
1) 즉시 기준본 복원
- `git restore .`로 작업트리 원복(세션 임시 산출물은 별도 관리).

2) 기능 수정 최소 재적용
- broad replace 금지.
- 메시지맵 1줄 + 핸들러 함수 추가처럼 "국소 패치"만 적용.

3) 문자열 이스케이프 검증
- `\n`이 실제 개행으로 깨지지 않았는지 검색 확인.
- `search_files`로 `Manual TIMER` 등 수정 문자열 주변 라인 재검증.

4) 인코딩 가드
- 파일별 인코딩 혼합 가능성을 전제로 `.h/.cpp` 각각 독립 검증.
- 인코딩 강제 변환 스크립트(전체 트리 대상) 금지.

권장 규칙:
- CP949 레거시에서는 "수정 최소화 + 즉시 검증"이 최우선.
- 회귀 시, 복원 후 다시 작은 패치 단위로 진행(한 번에 하나씩).
