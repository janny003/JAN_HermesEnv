# LAND8116 수동 스탑워치 통합 시 인코딩 가드레일 (세션 요약)

## 상황
- 목표: `LANDTestView`에 수동 스탑워치 버튼/모델리스 다이얼로그/Stop 시 리스트 기록 추가.
- 결과: `LANDTestView.cpp` 직접 패치 과정에서 인코딩 취약이 재발해 컴파일 붕괴.

## 재현된 증상
- 대표 에러: `C2059`, `C2001 (상수 줄바꿈)`, `C1057 (매크로 확장 EOF)`.
- 파일: `LANDTestView.cpp` (기존 CP949/깨진 문자열 혼재 구간).

## 안전 절차 (이 프로젝트 계열 권장)
1. 취약 파일(`LANDTestView.cpp`)에 대한 대규모 문자열 치환/메시지맵 블록 치환 금지.
2. 먼저 `git checkout -- <fragile-file>`로 즉시 원복해 빌드 정상성 회복.
3. 기능 추가는 가능하면 분리 `.cpp`(sidecar translation unit)로 구현하고, 헤더에는 선언만 최소 추가.
4. UI wiring(메시지맵/OnInitDialog 좌표 배치)은 한 번에 많이 바꾸지 말고, **한 지점 수정 후 즉시 빌드**.
5. 모델리스 다이얼로그는 `Create` 실패/닫힘 재오픈(`IsWindow`) 가드 필수.
6. 실패 시 계속 밀어붙이지 말고 즉시 원복 후 더 작은 패치 단위로 재시도.

## 검증 규칙
- 수정 직후마다 `MSBuild Debug|Win32` 실행.
- 실패 시 첫 블로킹 에러(파일:라인 + 에러코드)만 먼저 정리하고 다음 수정 진행.
- 최종적으로는 빌드 성공 상태를 우선 확보한 뒤 런타임 버튼 동작 검증 수행.
