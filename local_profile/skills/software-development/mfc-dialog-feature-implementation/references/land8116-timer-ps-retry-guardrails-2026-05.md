# LAND8116 타이머/전원 연동 재시도 가드레일 (2026-05)

## 상황 요약
- 목표: 타이머 다이얼로그 + 전원공급기 상태 연동을 테스트 화면에 연결.
- 제약: `LANDTestView.cpp`가 KR 인코딩 취약(C2001/C1057/C2059/C4819 이력)으로 작은 문자열/메시지맵 패치에도 붕괴 가능.

## 재현된 실패 패턴
1. `LANDTestView.cpp`에 타이머/PS 핸들러를 직접 삽입.
2. 즉시 컴파일러 다중 오류(C2001/C1057) 발생.
3. 같은 파일에서 추가 수정으로 복구 시도할수록 오류 확산.

## 효과적이었던 복구/진행 패턴
1. 즉시 원복: `git checkout -- LAND8116/LANDTestView.cpp`.
2. 빌드 게이트 복구: Debug|Win32 빌드가 `경고0/오류0`로 돌아올 때까지 기능작업 중단.
3. 기능 확장 재시도는 고위험 파일 직편집 대신:
   - 선언/정합은 `.h` 최소 수정,
   - 구현은 sidecar `.cpp`로 분리,
   - 런타임 연결점만 최소 터치(가능하면 ASCII-only).
4. 매 변경 후 즉시 빌드(한 번에 여러 패치 금지).

## 검증 규칙 (사용자 요구 반영)
- 사용자가 "두번씩 검증"을 요구하면 아래를 2회 연속 수행:
  1) Build
  2) 실행
  3) PROCESS_ALIVE 확인
  4) 종료
- 회차별로 결과를 분리 보고(PID 포함).

## 실무 메모
- `.rc` 측 RC2104 이슈는 `#pragma code_page(65001)` 적용 상태를 먼저 확인하고, 기능 변경 전에 리소스 파서 안정성을 보장한다.
- 기능 미완보다 빌드 붕괴가 우선순위 높음: 붕괴 시 즉시 안정 브랜치 상태로 되돌리고 재설계한다.
